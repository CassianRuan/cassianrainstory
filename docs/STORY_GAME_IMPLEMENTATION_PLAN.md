# 故事类单机网页游戏 AI 分步执行方案

## 1. 执行指令

本文件是后续 AI 开发代理的实施依据。执行时应：

1. 按阶段顺序实施，每完成一个阶段就运行对应验证，不跳过失败项。
2. 在不改变已确认产品规则的前提下，自主处理普通技术细节。
3. 缺少正式素材时先使用风格一致的临时素材或明确标注的占位素材，不能因此阻塞程序开发。
4. 不把故事内容写死在 React 组件中；故事、节点、朗诵、音效和小游戏均由 JSON 配置驱动。
5. 只有涉及付费服务、账号凭据、重大需求变化或无法安全推断的素材版权问题时才向用户确认。
6. 持续更新 `AI_USAGE_LOG.md`，如实记录实现、验证、失败和修正。

## 2. 产品目标

制作一个面向成年人的桌面端悬疑推理、恐怖惊悚类网页游戏。第一版为无服务器的单机体验，但部署到公网，任何玩家都能通过网址访问；每位玩家的进度仅保存在自己的浏览器中。

首发故事为《暴雨中的休止符》。故事采用固定线性剧情，以整张场景插画、自动朗诵、同步字幕、背景音乐、环境音效及三类小游戏完成演出。玩家必须完成当前小游戏才能继续剧情；失败后可无限重试。

## 3. 已确认需求

### 3.1 运行与平台

- 只适配电脑浏览器，优先支持最新版 Chrome 和 Edge。
- 使用静态前端架构，不需要登录、数据库或后端。
- 默认部署到 Vercel，先使用平台提供的公网域名。
- 自动保存当前故事、已到达的最远节点、当前设置和通关状态到 `localStorage`。
- 玩家可以回退到已经到达的剧情节点，但不能跳到尚未解锁的节点。
- 通关界面显示“故事结束”，提供“重新开始”按钮。

### 3.2 视觉和布局

- 视觉参考：暗色复古手绘、细腻线稿、低饱和棕灰色、暖黄局部照明、哥特悬疑绘本与电影分镜感。
- 使用 16:9 宽屏整张场景插画，通过淡入淡出、缓慢推拉、轻微摇移、闪电和暗场等效果表现镜头。
- 故事选择入口只保留为画面边缘的一个按钮；展开时使用轻量浮层，关闭后不遮挡游戏画面。
- 核心区域包括场景画面、字幕、演出控制、小游戏 HUD 和设置浮层。

### 3.3 自动演出与朗诵

- 剧情默认自动播放，无需玩家逐句点击。
- 使用浏览器 `SpeechSynthesis` 免费本地朗读，不接入付费语音 API。
- 旁白、里斯、埃文等角色分别匹配不同可用声音；找不到指定声音时按语言、性别倾向和音高规则降级。
- 通过角色独立的 `rate`、`pitch`、`volume`，以及标点、分句和停顿预处理，使朗读尽可能生动。
- 字幕与朗读同步显示，并突出当前句子。
- 支持暂停、继续，以及“跳过朗诵并进入下一剧情节点”。跳过演出不能跳过尚未完成的小游戏。
- 页面失焦、切换故事、回退节点或进入小游戏时必须停止当前语音，避免多段声音重叠。

### 3.4 音频

- 支持循环背景音乐和按节点触发的环境音、动作音效。
- 玩家可以分别调节朗诵、音效、音乐三路音量，并可静音。
- 浏览器自动播放受限时，首页使用“开始故事”按钮完成首次用户交互，再启动音频和朗诵。
- 音频切换使用短暂淡入淡出，避免突兀截断；同一环境音不得重复叠加。

### 3.5 三类小游戏的统一规则

- 小游戏出现时暂停剧情自动播放和普通节点计时。
- 每局限时 60 秒，允许 3 次错误。
- 时间耗尽或累计 3 次错误即失败，显示失败原因和“重新尝试”。
- 失败不改变剧情进度，可无限重试当前小游戏。
- 成功后记录完成状态，恢复剧情并解锁下一节点。
- 重新进入已经完成的历史小游戏时可以重玩，但不得清除最远解锁进度。

三类玩法：

1. **点击对应物品**：玩家在单张场景图中点击指定线索区域；点中后显示反馈和线索文案，点空白或错误物品计一次错误。
2. **找不同**：左右并排展示同一场景的两个版本；只允许点击右图，找到全部差异即成功，误点计错。
3. **键盘 QTE**：音符或按键提示沿轨道进入判定区；按下配置指定的键完成判定，支持 `A/S/D/F` 之外的按键；按错键或错过判定计错。

所有点击区域使用相对于原图宽高的 `0–1` 坐标，保证缩放后仍能正确命中。

## 4. 首发故事改编

保留《暴雨中的休止符》的主线：里斯在暴雨木屋调查钢琴家阿尔伯特之死，搜集闹钟、茶壶、弓箭和乐谱线索，与埃文对峙，弹奏钢琴打开暗格，最后在停电与袭击中反杀埃文。

建议节点顺序：

1. `arrival`：暴雨木屋外景与入室，介绍尸体、吊灯和花斑猫。
2. `spot-difference`：插入找不同。里斯先后观察“刚进入房间时的记忆画面”和“闪电再次照亮后的现场”，寻找被移动或异常的物品，为伪造现场建立依据。
3. `investigation`：点击物品小游戏，调查闹钟、茶壶、弓箭和乐谱。
4. `confrontation`：里斯与埃文对峙并完成推理陈述。
5. `piano-qte`：完成可配置键位的钢琴 QTE，打开书架暗格。
6. `attack`：埃文拔刀、猫炸毛、雷击停电和黑暗搏斗。
7. `ending`：闪电照亮尸体，猫恢复呼噜声，画面淡出并显示故事结束。

允许为互动节奏调整原剧本文字和节点位置，但不得改变核心凶手、推理结论和结局。

## 5. JSON 配置设计

故事入口使用 `public/stories/index.json`，单个故事使用 `public/stories/<story-id>/story.json`。实现前先为配置定义 TypeScript 类型，并使用 Zod 或等效方案在加载时校验。配置至少支持：

```json
{
  "schemaVersion": 1,
  "id": "pause-in-the-rain",
  "title": "暴雨中的休止符",
  "cover": "cover.webp",
  "startNodeId": "arrival",
  "characters": {
    "narrator": { "name": "旁白", "voiceHints": ["zh-CN"], "rate": 0.9, "pitch": 0.9 },
    "rees": { "name": "里斯", "voiceHints": ["zh-CN"], "rate": 0.86, "pitch": 0.78 },
    "evan": { "name": "埃文", "voiceHints": ["zh-CN"], "rate": 0.92, "pitch": 0.7 }
  },
  "audio": { "music": [], "sfx": [] },
  "nodes": []
}
```

节点统一字段包括 `id`、`type`、`nextNodeId`、`scene`、`lines`、`music`、`sfx` 和 `transition`。节点类型至少包括：

- `narrative`：场景、字幕、角色台词和自动演出。
- `object-hunt`：目标区域、线索反馈、时限和错误上限。
- `spot-difference`：左右图片及差异区域。
- `keyboard-qte`：键位序列、移动时长和判定窗口。
- `ending`：结局与重新开始。

禁止在公开配置中放置敏感信息。作为纯前端游戏，答案坐标会随静态文件下发，第一版接受玩家通过开发者工具查看答案的限制。

## 6. 建议技术栈和目录

- React + TypeScript + Vite
- Tailwind CSS 或 CSS Modules
- Zustand（演出、进度和设置状态）
- Howler.js（音乐和音效通道）
- Zod（故事 JSON 校验）
- Vitest + React Testing Library + Playwright

建议目录：

```text
src/
  app/
  components/
  engine/          # 节点调度、自动演出、进度控制
  speech/          # 声音选择、分句、暂停与取消
  audio/           # 音乐和音效管理
  games/           # object-hunt、spot-difference、keyboard-qte
  schemas/         # JSON schema 与 TypeScript 类型
  stores/
  styles/
public/
  stories/
    index.json
    pause-in-the-rain/
      story.json
      images/
      audio/
```

## 7. 分阶段实施

### 阶段 0：素材盘点和项目初始化

- 检查用户提供的图片、音乐、音效和剧本文本，建立“已有/缺少/临时替代”清单。
- 初始化 Vite React TypeScript 项目、代码规范、测试和 Vercel 配置。
- 保存视觉参考，仅将有明确使用授权的素材纳入产品。
- 创建环境无关的 README 和 `.gitignore`。

验收：`npm install`、`npm run dev`、`npm run build` 均成功；首页可在桌面浏览器打开。

### 阶段 1：配置模型与剧情引擎

- 实现故事索引、JSON 校验、节点加载和错误提示。
- 实现线性节点调度、最远节点限制、回退、跳过和结局重开。
- 实现 `localStorage` 版本化存档与损坏数据降级。
- 将完整剧本改写为节点配置，缺素材处先使用占位资源。

验收：不修改组件代码即可更换节点顺序和台词；刷新后恢复进度；不能前进到未解锁节点。

### 阶段 2：电影化画面和音频

- 实现宽屏场景、字幕、高亮、淡入淡出、推拉、摇移、闪电与暗场效果。
- 实现故事按钮、演出控制和三路音量设置。
- 实现 Howler 音乐/音效管理以及浏览器自动播放解锁流程。
- 验证场景切换不会残留上一节点音频。

验收：剧情可自动连续演出；暂停、继续、跳过、音量调整行为正确。

### 阶段 3：浏览器朗诵

- 实现角色到本地声音的匹配和稳定降级策略。
- 实现中文分句、停顿、语速和音高配置，以及字幕同步。
- 处理声音列表异步加载、暂停恢复、快速跳过、回退和组件卸载。
- 提供“测试角色声音”设置入口，允许玩家为各角色更换本机可用声音。

验收：旁白、里斯和埃文可听出差异；连续切换节点无叠音；无可用中文声音时给出清晰提示并允许仅字幕继续。

### 阶段 4：三类小游戏

- 先实现统一的 60 秒计时、3 次错误、失败重试和成功回调框架。
- 实现点击物品、找不同和键盘 QTE。
- 加入命中、误点、倒计时和成功/失败反馈音效。
- 为鼠标坐标换算、错误次数、QTE 判定和重试状态编写测试。

验收：三种游戏均必须成功才能首次前进；倒计时或第 3 次错误触发失败；重试后状态完全重置。

### 阶段 5：素材完善与整剧联调

- 将用户素材替换进对应节点；缺失内容生成与参考图一致的临时或正式候选素材。
- 逐一核对点击区域、差异区域和 QTE 节奏。
- 调整整段剧情至“纯演出约几分钟”，小游戏耗时不计入该目标。
- 校对字幕、朗读分句、角色归属、音效时机和音量平衡。

验收：从开场到结局完整通关一次，无断链、错图、叠音或无法继续的问题。

### 阶段 6：测试和公网部署

- 运行类型检查、lint、单元测试、生产构建和 Playwright 核心流程测试。
- 在 Chrome、Edge 的 1440×900 与 1920×1080 下人工冒烟测试。
- 部署到 Vercel，验证刷新深链、静态资源缓存、音频加载和 HTTPS 访问。
- 更新 README：本地运行、故事配置、素材替换、构建部署和已知限制。

验收：公网网址可访问；新浏览器可从头游玩；同一浏览器刷新后存档恢复；故事可完整通关并重新开始。

## 8. 测试清单

- [ ] 故事 JSON 缺字段或引用不存在节点时显示可理解的错误。
- [ ] 自动演出、暂停、继续和跳过不会产生重复推进。
- [ ] 字幕与当前朗读句一致，角色切换时正确换声。
- [ ] 切换标签页、回退节点和切换故事时无叠音。
- [ ] 三路音量独立工作并被保存。
- [ ] 三种小游戏均执行 60 秒、3 次错误和无限重试规则。
- [ ] 点击区域在不同桌面分辨率下准确。
- [ ] QTE 按错键和漏按均正确计错。
- [ ] 未完成小游戏不能解锁后续节点。
- [ ] 已解锁节点可以回退，未解锁节点不可选择。
- [ ] 刷新、关闭后重开能够恢复进度和设置。
- [ ] 通关后能重新开始，且不会保留本轮小游戏临时状态。

## 9. 开工所需素材

用户可逐步提供，无需一次齐全：

- 各剧情节点的 16:9 场景插画。
- 找不同所需的左右两图，以及差异位置说明或标注图。
- 点击物品场景图，以及四个目标物品的位置。
- 背景音乐、雨声、雷声、脚步、猫叫、钢琴、机关和搏斗等音频。
- 封面图和故事按钮缩略图。

素材建议使用 WebP/PNG 图片及 MP3/OGG 音频。所有素材必须确认拥有网页发布权限；未提供的素材由 AI 生成或使用可替换占位资源，并在清单中明确标记来源与状态。

## 10. 明确限制

- 浏览器本地声音因操作系统而异，无法保证所有玩家听到完全相同的音色和情绪效果。
- `SpeechSynthesis` 的精确字级时间信息兼容性有限，第一版以句级字幕同步为主。
- 静态单机版的 JSON 和小游戏答案可被技术型玩家查看，不提供防作弊能力。
- 浏览器清理站点数据、更换浏览器或设备后，存档不会自动迁移。
- 第一版不适配手机，不包含账号、云存档、多人功能、后台编辑器或付费服务。

## 11. 完成定义

只有同时满足以下条件才算 MVP 完成：

1. 《暴雨中的休止符》可从开场完整游玩至结束。
2. 自动朗诵、字幕、音乐、音效和三类小游戏全部实际运行。
3. 所有游戏门槛、失败重试、回退和存档规则符合本文。
4. 自动测试和目标桌面浏览器人工冒烟测试通过。
5. Vercel 公网地址可访问，并提供本地运行及新增故事配置说明。
