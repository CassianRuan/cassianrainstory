# 雨声无缝循环裁剪设计

## 目标

移除持续雨声末尾接近静音的淡出段，并让音频循环回开头时自然衔接，不出现空档、爆音或明显音量跳变。

## 素材现状

- 项目实际引用 `public/stories/pause-in-the-rain/audio/ambience/ambience-rain.wav`，没有引用同名 MP3。
- 文件为 16-bit PCM、双声道、22050 Hz，总长约 34.01 秒。
- 0–31.75 秒的雨声平均响度基本稳定在约 -24 至 -26 dB。
- 约 32 秒开始淡出；33.50–33.75 秒降到约 -48.6 dB，33.75 秒后低于约 -66 dB，形成可听见的循环空档。

## 已批准方案

1. 将有效素材截到 32.00 秒，删除之后约 2.01 秒的淡出和近静音内容。
2. 使用 1.50 秒等功率交叉淡化，将裁剪后尾部与原开头混合。
3. 输出从原素材 1.50 秒处开始，接续到 30.50 秒，再附加 30.50–32.00 秒尾部与 0–1.50 秒开头的交叉淡化段。这样文件末端已经过渡到开头的 1.50 秒位置，循环回文件起点时能延续该位置之后的雨声。
4. 输出预计约 30.50 秒，保留原 WAV 路径、PCM 位深、声道数和采样率，因此故事 JSON 与播放器代码无需修改。
5. 如果交叉淡化使峰值超过 95% PCM 满幅，对完整输出应用统一安全缩放；不使用硬削波。

## 处理工具

新增一个确定性 Node.js 脚本读取 RIFF/WAVE 块，验证输入为 16-bit PCM，并接受以下参数：

```text
node scripts/make-seamless-wav.mjs <input.wav> <trim-end-sec> <crossfade-sec>
```

本次执行参数固定为：

```text
node scripts/make-seamless-wav.mjs public/stories/pause-in-the-rain/audio/ambience/ambience-rain.wav 32 1.5
```

脚本在原路径写回处理结果，并输出原时长、新时长、交叉淡化时长、处理前后峰值和安全缩放系数，便于复查且不会依赖外部付费软件。

## 验证标准

- 文件仍能识别为 RIFF/WAVE、16-bit PCM、双声道、22050 Hz。
- 新时长约为 30.50 秒，允许一个采样帧以内的舍入差异。
- 末尾不存在连续近静音段；最后 0.25 秒 RMS 不低于 -35 dB。
- 处理后峰值不超过 95% PCM 满幅，无样本硬削波。
- 对循环边界前后各 100 ms 的 RMS 进行比较，差异不超过 3 dB。
- 故事配置测试、TypeScript、Vitest 和生产构建全部通过。
- 在真实 Chromium 中确认雨声保持播放超过两个完整循环，循环边界前后播放器始终为播放状态，且没有音频加载失败。

## 不在本次范围内

- 不更换为 MP3、OGG 或其他音频格式。
- 不改变雨声配置音量 `0.85`、朗诵时 55% 动态压低或玩家音量设置。
- 不调整背景音乐、跑酷音效和其他场景声音。
