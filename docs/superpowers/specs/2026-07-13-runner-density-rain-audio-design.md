# 跑酷障碍密度与雨声音频调整设计

## 目标

降低汽车跑酷的障碍密度，并确保玩家从办公室开场开始就能清楚听见持续雨声。雨声需要贯穿所有雨景，同时在角色朗诵期间自动降低，避免遮盖台词。

## 已确认需求

- 跑酷障碍生成间隔从前期约 2.5 秒逐渐缩短到后期约 1.8 秒。
- 每组障碍仍最多占据两条车道，不能完全堵死道路。
- 继续使用现有 `ambience-rain.wav`，不更换为 MP3。
- 增强雨声音频素材本身的响度，并将故事配置音量设置为 `0.85`。
- 增强后的雨声用于所有引用 `rain` 环境音的剧情节点。
- 朗诵开始后，雨声平滑降低到正常播放增益的 55%；朗诵结束、跳过或取消后平滑恢复。
- 保留玩家存档中的音效音量和静音选择，不重置个人设置。

## 当前问题与根因

故事开场节点已经声明 `ambience: "rain"`，音频路径也指向有效的 WAV 文件。素材为 16-bit、双声道、22050 Hz PCM，但测得平均响度约为 `-28.7 dB`。当前素材音量 `0.62` 还会乘以默认音效通道音量 `0.72`，最终增益约为 `0.45`，导致雨声在扬声器或朗诵同时播放时很难听见。

此外，音频是在 React 状态变化后的 effect 中启动，而不是直接在“开始故事”点击处理函数中解锁。浏览器的媒体自动播放策略可能使首次播放请求失去用户手势上下文。实施时需要显式解锁音频上下文，并验证环境音确实进入播放状态。

## 设计方案

### 跑酷难度

保持现有三车道、随机生成、速度渐增、最多封锁两条车道和碰撞规则不变，仅将 `night-drive` 的障碍间隔参数调整为：

- `spawnIntervalStartMs: 2500`
- `spawnIntervalEndMs: 1800`

现有的 ±10% 随机扰动继续保留，避免障碍出现节奏过于机械。难度函数和道路速度曲线不做额外改动。

### 雨声素材

对现有 WAV 做确定性的响度增强，保留格式、采样率、声道和循环用途。处理后检查峰值，避免削波和爆音。故事 JSON 中 `rain` 的素材音量改为 `0.85`。

音效通道音量仍由玩家的“音效”滑块控制。静音或音效音量为零时，程序必须尊重玩家选择，不强制播放。

### 首次播放解锁

在玩家点击“开始故事”或“继续故事”的同一个用户手势中恢复 Howler 音频上下文，然后进入故事。节点音频同步逻辑仍负责选择雨声、音乐和循环音效，避免在界面组件中复制资源选择规则。

如果恢复失败，保留后续用户交互再次恢复的能力，不让异常阻止剧情界面加载。

### 朗诵动态压低

音频管理器提供环境音压低状态，而不是由各个字幕组件直接操作 Howler 实例：

- 朗诵开始：在短渐变时间内将环境音目标增益乘以 `0.55`。
- 每句之间继续保持压低，避免雨声音量反复起伏。
- 节点朗诵全部结束、玩家跳过、取消朗诵或离开节点：平滑恢复到当前玩家音效设置与素材音量共同计算出的正常增益。
- 暂停时不改变玩家的持久化音量设置。

动态压低只作用于 `ambience` 通道，不改变背景音乐、引擎、湿路、雨刷及一次性音效的现有混音关系。

## 组件边界与数据流

1. 开始按钮在用户手势内请求恢复音频上下文，再更新游戏开始状态。
2. `App` 根据当前节点将 `rain` 同步到环境音通道。
3. `AudioManager` 将玩家音效音量、素材音量和当前压低系数组合为最终环境音增益。
4. 朗诵播放器在整段朗诵开始和所有退出路径上切换压低状态。
5. 节点切换仍由既有逻辑更新音频，不引入第二套播放器。

## 错误处理

- 音频上下文恢复失败时不阻塞故事，只记录播放失败并允许下一次交互重试。
- 雨声素材加载失败时，其他音乐、音效和剧情继续运行。
- 朗诵被跳过、取消或组件卸载时必须释放压低状态，防止后续雨声永久偏小。
- 多个连续朗诵事件使用单一压低状态，避免渐变叠加。

## 验证标准

- 配置测试确认跑酷间隔为 2500→1800 毫秒，且公平生成规则仍保证至少一条可通行车道。
- 音频单元测试确认环境音正常增益、55% 压低增益及恢复增益计算正确。
- 组件测试覆盖开始按钮触发音频解锁，以及朗诵正常结束、跳过和卸载后的恢复。
- 音频处理后无削波，文件仍为可循环播放的有效 WAV。
- TypeScript、Vitest 和生产构建全部通过。
- 真实桌面浏览器从首页点击“开始故事”后，办公室第一幕能够听见清晰雨声；朗诵期间雨声降低但仍可察觉，朗诵结束后平滑恢复。
- 跑酷浏览器验收确认障碍明显疏于当前线上版本，仍会随进度适度增加，并且没有三车道同时封死的情况。

## 不在本次范围内

- 不新增独立的“环境音”设置滑块。
- 不替换雨声素材内容或转为 MP3。
- 不调整跑酷速度、碰撞次数、无敌时间、终点时长或障碍种类。
- 不改变背景音乐及其他场景音效的音量。
